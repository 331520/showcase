In case if you wanted to set the displaying through RELOAD of the page, but for some errors like ViewExpiredException you would demand RELOAD, you can define it with various logs with adding a definition of the attribute TYPE. If the type is not assigned, define DEFAULT behaviour, if the type is defined, you determine a specific behaviour for handling only for one type of error/exception.

An example frequently displays the utilized definition, where after the error ViewExpiredException is in demand of a RELOAD and in other error the RELOAD will not be necessary. ${showcase:getFileContent('/sections/ajaxerrorhandler/example-ajaxerrorhandlerType.xhtml')} ${showcase:getFileContent('/org/primefaces/extensions/showcase/controller/AjaxErrorHandlerController.java')}
